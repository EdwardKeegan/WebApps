You're Going to Love This Web Site!

This example shows a background image instead of just a background color. Things can get a bit tricky when using a background image because you never know how tall and/or wide the browser is going to be. It's probably best to use background-attachment:fixed (see the body CSS rule). That way the image does not have to try to cover a really tall skinny page (several viewports tall) like you might get from a mobile device - the background just has to cover a single viewport. It's a neat effect. Scroll down and note that the background image does not scroll.

Also, use background-size:cover to be sure that the background image always covers all the area.

It's very important to have the background image have low contrast so that the words on top can still be read with no difficulty. You can easily edit a background image using MSPaint or Paint.Net (free to download). Just reduce contrast then lighten if you will use dark text or darken if you will use light text. You probably want a fairly large sans-serif font for the text that's on top of a background image. You may want to increase line-height and add letter-spacing for even more readability.
