Web Dev Tips

  • Text: Make sure that all text (including link tags <a>) is ALWAYS easy to read.
    • Make the text have enough contrast with its background.
    • Make the text large enough.
    • Use line-height for spacing between lines. You can also use letter-spacing for a stylized look within an important word.
    • Use padding and/or margin to prevent text from being too close to any visible edge.
    • Use "rem" for measuring font-sizes and vertical spacing (like padding/margin top/bottom).
      • "rem" is a newer measurement unit that means the normal font-size of the root element of the viewport. It scales well (desktop/mobile).
  • Fluid Layout (width): narrow the browser to test the layout at different widths. It should always wrap nicely filling out all the horizontal space (no wasted space, no horizontal scroll bar).
    • Use percentage for specifying widths and horizontal spacing (like padding/margin left/right).
      • Remember that with "box-sizing:border-box" (which is recommended), div widths include border and padding but not margin.
      • Use math to allocate the 100% width.
  • Don't mix measurement units (e.g., pixels and rem) on any dimension (e.g., width or height) since this can cause alignment problems on different sizes of viewports.
  • Vertical Layout: make sure your page would look good with a lot of content and with a little content. We will be writing single page applications and the content will change depending on what the user clicks on.
  • Code Indentation: always indent your code while programming indented using Alt-Shift-F or "Source - Format" from the NetBeans menu.
