This version has a media query that makes the web page "responsive" meaning that it looks good regardless of the width of the viewport, be it desktop or mobile device.

Narrow your browser window until you see a change in the layout. The nav bar should go under the title and the top margin of the content area gets taller so that the top of the content is not hidden under the taller titleNav.

Although you may find examples that use pixels as the unit of measurement for the media query min-width, I recommend that you use "rem" instead (and I looked into it from here). 

   @media only screen and (min-width: 40rem) { 

       /* ... */

   }
Since you have specified your title and nav bar link text using rem measurements, specifying the media query min-width also with rem will provide consistent results when you narrow your browser.
