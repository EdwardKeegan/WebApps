CSS

CSS stands for Cascading Style Sheets. It's the language we use to style HTML elements.

CSS Selectors

So that we can specify which elements we want to style, we can add id and/or class attributes to the HTML elements. An id is supposed to be unique (one only) to the page, whereas a class may exist more than once on the page, for example: 

    <div id='nav'>
        Nav Bar
        <div class='navLink'><a href="">link1</a></div>
        <div class='navLink'><a href="">link2</a></div>
        <div class='navLink'><a href="">link3</a></div>
    </div>

As shown in the CSS figure above, each rule-set begins with a selector. There are several ways to select which HTML elements are to be styled.

  • html tag selectors, e.g, body { ... } or p { ... }
  • id selectors, e.g, #nav { ... }
  • class selectors, e.g, .navLink { ... } -- note the period before "navLink"
  • pseudo selectors, for example:
    • when the user hovers over class navLink: .navLink:hover { ... }
    • to style links before they are clicked a:link { ... } or
    • to style links after they are clicked a:visited { ... }.
  • compound selectors, e.g., links inside of #nav: #nav a { ... }

CSS Properties

CSS properties can add styling for:

  • fonts: font families, colors, font size, line height, letter spacing, alignment
  • box styling (e.g., to style HTML div tags, which are essentially "boxes"): margin, padding, borders, dimensions, background colors and images
  • positioning: static, fixed, relative, absolute
  • transitions: "slow down" the changing of various CSS properties (provides an animated look)
  • Media Queries: specify different layouts depending on the width of the viewport (e.g., desktop or mobile)

CSS Measurements

There are a lot of different measurements, some which are absolute (like inches) and some (like pixels) where the size will vary substantially depending on the resolution of the monitor. You don't really want to mix your measurements either, or your layout may not align properly, depending on the viewport. So, here are some recommendations:

  • For heights including font-size, top/bottom padding and margin, I recommend you use "rem", a newer measurement that's equal to the font-size of the root element. This measurement scales well on various viewports.
  • For widths, its best to use percentages so that our layouts are fluid (use all available horizontal space and wrap nicely depending on the width of the viewport). If we used something like pixels to width measurements, then we'd typically have wasted width or find the layout wider than the viewport and have to to use the left/right scroll bar to read all the text -- neither of which is good.

Where to Put CSS

We can put CSS rule sets in three places:

  • inline styling (no reuse): rules are placed right in a specific HTML tag, like this: 
        <p style="background-color:pink; font-weight:bold;">This is a paragraph.</p>
  • internal style sheet: rules are placed inside of a <style> tag inside the <head> section of a web page. Right click and View Source on this page and you'll see that we used an internal style sheet to style this page.
  • external style sheet: rules are placed in a separate ".css" file which is referenced by a <style> tag that's inside the <head> section of a web page.

Comments in CSS must be of this style: /* ... */
