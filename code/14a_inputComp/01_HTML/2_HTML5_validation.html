HTML5 Features for Validating Input

There is no JavaScript on this page!   Any validation you see is implemented using (the newer) HTML5 validation features. When the user clicks a form's submit button, all the inputs of the form are validated (by the browser, according to the validation rules specified in the HTML tags).

HTML5 "input validation bubbles" only show the first error. If the user fixes the first error, they might see the next error. But finally, when all errors have been fixed, the inputs are sent to the server side program as specified in the form's action attribute.

Like in the previous example, all forms on this page send their data to my generic server side program "receiveData.jsp". "receiveData.jsp" is a generic server side program that just lists the names and values of all the inputs sent to it.
