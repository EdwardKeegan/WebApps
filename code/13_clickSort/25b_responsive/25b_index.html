Responsively Designed HTML Table

To test this, narrow your browser and then make it wider.

  • At narrow widths, you'll see only two "mobile" columns. Some of the object properties are displayed in the first mobile column and the remaining properties are shown in the other mobile column. Properties are displayed vertically (since cell phones are tall and skinny).
  • With a wide viewport, the two mobile columns are hidden, but the desktop columns are revealed. Like the "MakeTableBetter" code examples, there is one desktop column per property of the objects in the object list.
  • If you inspect this page, you'll see that the mobile columns and the desktop columns coexist within the rows of the HTML table (the header row and/or any data row). The CSS (mobile first) hides the desktop columns and shows the mobile columns. Then the CSS media query does the reverse (shows the desktop columns and hides the mobile columns).
  • Whether you are in Mobile Layout or Desktop Layout, you should be able to scroll up and down within the HTML table without loosing the column headings (which have been styled as "sticky"). This helps save space in Mobile Layout because you do not have to repeat column heading values for each data row (as I have seen others do, out on the web). It's also helpful for Desktop Layout when the number of data rows is large.
