This achieves User Interface reuse but without routing. Instead of having an "href" attribute, the links in the nav bar have an onclick function that invokes the correct JavaScript component (function) to inject the right content. Note that we are following our strict naming convention that every JS file is named for the single function (or single object) defined in that file. Each of these two functions, just specifies some HTML code, then "injects" that code into the content area, in this example, the div with id "view".

The downside to this simple approach is that you never see any changes in the URL and the user cannot use the browser's navigation buttons (forward/back).
