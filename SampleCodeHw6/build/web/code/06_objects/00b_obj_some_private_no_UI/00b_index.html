What is JavaScript Closure?

If you look at the MakeCar function through the eyes of a Java programmer, you might ask "Why declare condition and price as local variables when these variables will just be destroyed when the MakeCar function finishes running?"

JavaScript variables are not destroyed when the function that created them goes out of scope. JavaScript variables (like Java Static variables) retain their values even after the function that created them is "closed" -- I think this is why they call it "JavaScript Closure".
