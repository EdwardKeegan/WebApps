This example shows the "mechanics" of how to pass parameters in an object instead of in a parameter list. For this example, I did not bother putting the JS functions in external files, but the technique is the same. By using parameter objects, the code becomes more self documenting (you do not have to "look up" what is the first parameter, the second, etc). It is also easier to deal with optional parameters. If the calling code does not supply a value for some property of the parameter object, the function can supply a default value. The possible "down side", is that you must spell the parameter property names correctly or else they will not be picked up by the function being called (but this is probably no worse than having to know the order of parameters in a parameter list).
