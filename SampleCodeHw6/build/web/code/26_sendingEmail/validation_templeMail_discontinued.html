When the user clicks the Submit button,

  • If any input does not pass HTML5 validation rules, the user will see an HTML5 "bubble" validation pop up (of the first error only).
  • If all input passes HTM5 validation rules, then all the input data is sent to the server side program that is specified by the "action" attribute of the form tag. In this case, it invokes a Temple Mail (server side program) that sends all the user's input values to sallyk@temple.edu.
    • NOTE: Temple recently retired the server side program that used to send mail, but you can find a hoster that would provide similar functionality.
